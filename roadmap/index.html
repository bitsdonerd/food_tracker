Design Patterns
3 Padrões para 3 Problemas
Cada pattern resolve um problema específico do domínio de restaurante
FACTORY
PedidoFactory
Problema: Marmita e PF têm regras de preço completamente diferentes. Espalhar esse if/else nos componentes React seria um desastre de manutenção.

Solução: A Factory centraliza a criação. O componente não sabe como o preço é calculado — ele só chama PedidoFactory.criar().
// domain/PedidoFactory.ts export class PedidoFactory { static criar(tipo: TipoPedido, dados: PedidoInput) { switch (tipo) { case 'MARMITA': return new Marmita(dados) // 1 carne = R$18 | 2 carnes = R$20 case 'PF': return new PratoFeito(dados) // 2 carnes OU 1 carne dobrada default: throw new Error('Tipo inválido') } } } // No API Route — limpo, sem if/else: const pedido = PedidoFactory.criar(body.tipo, body) await prisma.order.create({ data: pedido.toDatabase() })

BUILDER
PedidoBuilder
Problema: Um pedido pode ter arroz, feijão, salada, 2 carnes diferentes, sem cebola, extra farofa... Construir esse objeto em um único new() é impossível de ler.

Solução: O Builder monta o pedido passo a passo. "Completo" chama todos os métodos de uma vez. "Custom" chama só o que o garçom selecionar.
// domain/PedidoBuilder.ts export class PedidoBuilder { private itens: Item[] = [] setCompleto() { // Adiciona todos os acompanhamentos padrão this.itens = [...ACOMPANHAMENTOS_PADRAO] return this // chainable } addCarne(carne: string) { this.itens.push({ tipo: 'CARNE', nome: carne }) return this } addAcompanhamento(item: string) { this.itens.push({ tipo: 'ACOMP', nome: item }) return this } build(): PedidoItems { return { itens: this.itens } } } // Uso: garçom clica "Completo" const itens = new PedidoBuilder() .setCompleto() .addCarne('Frango Grelhado') .build()

OBSERVER
Pusher Channels
Problema: A cozinha precisa ver o pedido imediatamente sem dar F5. Polling a cada N segundos desperdiça recursos e adiciona latência desnecessária.

Solução: A cozinha se inscreve no canal Pusher (Observer passivo). Quando um pedido é salvo, a API dispara o evento. A cozinha é notificada em < 500ms.
// app/api/orders/route.ts — Sujeito export async function POST(req: Request) { const pedido = await prisma.order.create({...}) // Dispara evento para todos os Observers await pusher.trigger( 'kitchen-channel', // canal 'new-order', // evento { pedido } // payload ) return NextResponse.json({ pedido }) } // app/dashboard/page.tsx — Observer const channel = pusher.subscribe('kitchen-channel') channel.bind('new-order', (data) => { setPedidos(prev => [data.pedido, ...prev]) playAlertSound() // 🔔 })

// domain/Marmita.ts export class Marmita { private proteinas: string[] = [] private acompanhamentos: string[] = [] constructor(private data: MarmitaInput) {} calcularPreco(): number { const qtdProteinas = this.proteinas.length if (qtdProteinas === 1) return 18.00 if (qtdProteinas === 2) return 20.00 throw new Error('Marmita: 1 ou 2 carnes apenas') } toDatabase() { return { type: 'MARMITA', totalPrice: this.calcularPreco(), items: [...this.proteinas, ...this.acompanhamentos] } } } // domain/PratoFeito.ts export class PratoFeito { calcularPreco(): number { return this.data.precoFixo ?? 25.00 } getProteina() { // 2 carnes normais OU 1 dobrada (isDoubled) if (this.data.carneDobrada) { return [{ nome: this.data.carne, isDoubled: true }] } return this.data.carnes.map(c => ({ nome: c, isDoubled: false })) } }
